# Behind The Eyes — Demo Flow & Experience Guide

## Core Concept

Behind The Eyes transforms passive cinema into an interactive psychological experience.

Instead of summarizing videos or chatting with movies, the system allows viewers to pause a scene, interrogate a character, and reveal the hidden thoughts they never said aloud.

The experience should feel cinematic, immersive, and psychologically intimate — not like a chatbot UI.

---

# Intended Emotional Experience

The viewer should feel:

* like they are interrupting reality
* like the character is confessing something hidden
* like cinema itself became psychologically explorable
* like the fourth wall is quietly breaking

The experience is intentionally slow, dramatic, and minimal.

Silence and pacing matter.

Do not rush transitions.

---

# UI Workflow

## 1. Landing State

The application opens directly into a dark cinematic interface.

Visible UI:

* Video URL input
* Show / Series input
* Character chips/buttons
* Embedded video player

The interface should feel closer to a streaming platform than an AI dashboard.

No terminal-style visuals.
No visible debugging.
No multiple panels.

---

# 2. Load Video

Paste a YouTube clip URL : https://www.youtube.com/watch?v=GO8LiD-gH70&pp=ygUKc2NhbSAxOTkyXA%3D%3D

Examples currently supported well:

* Scam 1992 (Enter in show/series tab / then click 'Set')

Click:
`Load`

Then:

* the video loads
* characters appear automatically as selectable chips
* the player becomes the center of attention

The UI should communicate:
“This scene has now been understood.”

---

# 3. Select Character

Click a character chip.

Examples:

* Harshad Mehta

This opens the interrogation modal.

The modal should feel elegant and restrained — almost like opening a confidential file.

---

# 4. Ask Psychological Question

Type a psychologically revealing question.

* “What makes people trust you?”
Avoid meme/joke prompts during official demo.

The strongest responses happen when:

* the question targets ego
* fear
* manipulation
* insecurity
* ambition
* hidden intent

---

# 5. Thinking State

After clicking `Ask`:

This moment is IMPORTANT.

Do not skip quickly.

This pause creates anticipation and sells the illusion that the character is internally processing the question.

Silence here improves emotional impact.

---

# 6. Character Response

The generated response appears above the video player.

The response should feel:

* emotionally sharp
* psychologically revealing
* consistent with the character
* intimate
* like hidden internal monologue

The system uses:

* VideoDB transcript retrieval
* scene indexing
* surrounding dialogue context
* multimodal scene understanding
* prompt-grounded character cognition

The response is NOT intended to feel like generic roleplay.

It should feel like:
“the thought the character would never say aloud.”

---

# 7. Continue Playback

The video remains directly beneath the response.

This creates the illusion that:
the viewer now understands the scene differently after hearing the hidden cognition.

The audience should mentally re-interpret the scene after the confession.

That emotional reframing is the core magic of the product.

---

# Demo Delivery Style

The demo should be narrated slowly and cinematically.

Avoid:

* technical buzzword overload
* rapid feature listing
* overexplaining implementation

Focus instead on:

* immersion
* psychology
* emotional reframing

Good narration tone:
“Characters lie to each other.
But they hide even more from the audience.”

Then:
“Behind The Eyes lets viewers interrogate the silence between dialogue.”

---

# Important Framing

Do NOT frame this as:

* “chatting with movie characters”
* “AI movie assistant”
* “video chatbot”

The stronger framing is:

“Interactive cognitive layer for cinema.” / Breaks the 4th wall of cinema

Or:

“A system that reveals hidden character psychology from scene context.”

---

# Technical Notes

VideoDB powers:

* video ingestion
* transcript extraction
* scene indexing
* contextual retrieval
* multimodal understanding
* character grounding workflows

The architecture intentionally keeps the experience centered around a single cinematic player to preserve immersion.

---


# Final Intended Feeling

The viewer should leave thinking:

“I was not talking to the character.”

“I was listening to the thought they were trying to hide.”
